An open-source validation engine that checks LLM-generated chemistry, physics, mathematics, and statistics against fixed external authorities. When the math is wrong, the engine says so — regardless of what was attested.

The shape of the problem

Current AI verification scales as O(n²): every model polls every other model, cost grows with participants, and reliability is bounded by the least reliable node. Authority-based validation scales as O(1): every claim is checked against a fixed external standard. Cost is constant. The standard does not drift with participant count.

An example exchange

User
Compute the two-tailed p-value for n=30, mean₁=5.0, mean₂=4.0, sd=1.0. Is it significant at α=0.05?
LLM (without engine)
The p-value is approximately 0.0001. That's a confident-sounding number — but it's wrong.
LLM + Concordance Engine
Calls verify_statistics_pvalue. Engine recomputes from scipy: p ≈ 0.0003. LLM corrects itself: yes, significant at α=0.05; the correct p is 0.0003, not 0.0001.

Try it now

The engine ships with an MCP server. Connect it to Claude Desktop in two steps:

pip install -e ".[mcp]"
concordance-mcp

Then in claude_desktop_config.json:

{
  "mcpServers": {
    "concordance-engine": { "command": "concordance-mcp" }
  }
}

Restart Claude Desktop. The verifier tools now appear in the tool picker. Full install guide →
