An open-source validation engine and MCP server that checks LLM-generated scientific claims against fixed external authorities — not other models. When the math is wrong, the engine says so.

Get started on GitHub How it works

The shape of the problem

Current AI verification scales as O(n²): every model polls every other model. Cost grows with participants, and reliability is bounded by the least reliable node. Authority-based validation scales as O(1).

Without Concordance

O(n²)

Model polls model. Costs compound. The network is only as reliable as its weakest node. Consensus can be wrong, confidently.

With Concordance

O(1)

Every claim checked against a fixed external standard. Cost is constant. The standard does not drift with participant count.

User

Compute the two-tailed p-value for n=30, mean₁=5.0, mean₂=4.0, sd=1.0. Is it significant at α=0.05?

LLM without engine

The p-value is approximately 0.0001. Confident-sounding — but wrong.

LLM + Concordance Engine

Calls verify_statistics_pvalue. Engine recomputes via scipy: p ≈ 0.0003. LLM self-corrects: significant at α=0.05; correct p is 0.0003, not 0.0001.

Seven domains. Twelve verifiers.

Get started in two steps

Runs locally as an MCP server. Connects to Claude Desktop, Cursor, or any MCP-compatible client.

1. Install

Clone the repo and install with pip.

terminal 
# Clone and install
git clone https://github.com/matharrismma/Lighthouse
cd Lighthouse
pip install -e ".[mcp]"

# Start the MCP server
concordance-mcp

2. Connect

Add to your claude_desktop_config.json.

claude_desktop_config.json 
{
  "mcpServers": {
    "concordance": {
      "command": "concordance-mcp"
    }
  }
}
