What the benchmark measures

Each of the 171 items gives the model a domain-specific question requiring a precise numeric answer, a yes/no classification, or a short string. The question instructs the model to call the relevant concordance-engine verifier and reply with only the result.

Scoring is exact: numeric answers must match the engine-computed ground truth within a domain-appropriate relative tolerance (typically 0.1%). Classification and string answers require an exact match (case-insensitive, punctuation-normalized).

The benchmark intentionally uses questions where the right answer requires computation — not recall. Statistics p-values, radioactive decay constants, Carnot efficiency, Shannon entropy, critical path lengths, wave speeds. These are the questions LLMs hallucinate confidently without external verification.

The statistics result

Statistics was the original headline: Claude Haiku 4.5 scores 0% on p-value arithmetic from raw inputs without tools — 100% with verify_statistics. The engine was built to close exactly that gap. Across 57 domains the pattern holds: the verifiers carry the weight that language models cannot.

Reproducing the benchmark

git clone https://github.com/matharrismma/Lighthouse
cd Lighthouse
pip install -e ".[mcp]" anthropic

export ANTHROPIC_API_KEY=sk-ant-...

# Rebuild the 171-item dataset
python eval/benchmark/build_items_extended.py

# Run with tools (full benchmark, ~8 min on Haiku)
python eval/benchmark/run_extended.py --tools-only

# Run both modes for a side-by-side delta
python eval/benchmark/run_extended.py

Results write to eval/benchmark/results_ext_*_tools.jsonl alongside a printed per-domain summary. The delta report shows exactly which items the engine fixed and which (if any) it broke.

Polymathic synthesis benchmark

A second benchmark tests cross-domain synthesis: given a multi-domain situation, the engine must assign one of five verdicts — CONCORDANT, DISCORDANT, MIXED, QUARANTINE, or OUT_OF_SCOPE — using axis-weighted scoring across all fired verifiers.

The 25-point gap is structural: (1) p-value precision requires calibrated numerical verification tools cannot hallucinate; (2) MIXED vs DISCORDANT distinction requires axis-weighted mismatch fraction computation that language models cannot reliably perform across five domains in their heads.

What this benchmark cannot prove

171 items across 57 domains is a meaningful dataset — not a definitive one. It cannot:

  • Prove the engine helps with frontier models that may handle more computation natively.
  • Prove the engine helps in agentic workflows where tool latency is a constraint.
  • Prove correctness in open-ended analysis tasks — the benchmark tests deterministic computation, not judgment.
  • Prove the engine catches errors in real published work — that is a separate, ongoing effort.

It proves what it proves: 63 verifiers produce correct, deterministic answers on a self-validating dataset, and a model with access to those verifiers scores 99.4% across the 57 benchmark domains. The one residual miss is a model token-budget cap, not an engine error. That is the foundation.
