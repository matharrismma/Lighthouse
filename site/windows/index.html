1. Receive an adapter from the Mac

The Mac sends with tailscale file cp adapter.tgz harrismotors:. On Windows, fetch from the Tailscale inbox and extract into the canonical data/models/mlx/ tree.

cd "$env:USERPROFILE\OneDrive\Documents\Claude\Projects\Lighthouse"
mkdir -Force data\models\mlx | Out-Null

# Pull whatever's in the Tailscale inbox into a staging dir
$staging = "data\inbox\mac"
mkdir -Force $staging | Out-Null
tailscale file get $staging
Get-ChildItem $staging

# Extract — replace  with the run_id the Mac printed
$RUN_ID = "mlx-Mistral-7B-Instruct--20260527-171852"
$dest   = "data\models\mlx\$RUN_ID"
mkdir -Force $dest | Out-Null
tar -xzf "$staging\adapter.tgz" -C $dest
Get-ChildItem $dest\adapters | Format-Table Name, Length

2. Register the model

Adds the adapter to data/models/registry.json with provenance (base, corpus SHA, run id, backend).

cd "$env:USERPROFILE\OneDrive\Documents\Claude\Projects\Lighthouse"
$RUN_ID = "mlx-Mistral-7B-Instruct--20260527-171852"
$CORPUS = "data\training_corpus\corpus-gated-generation-20260526-103857.jsonl"

python tools/model_registry.py register `
    --base mlx-community/Mistral-7B-Instruct-v0.3-4bit `
    --adapter "data/models/mlx/$RUN_ID/adapters" `
    --backend mlx `
    --corpus $CORPUS `
    --run $RUN_ID

3. Benchmark against the Anthropic baseline

Runs the same eval set the Anthropic baseline ran on (data/eval/prompts_v1.jsonl). Publishes a new aggregate to site/benchmark/latest/aggregate.json, which the public benchmark page picks up.

cd "$env:USERPROFILE\OneDrive\Documents\Claude\Projects\Lighthouse"
$RUN_ID = "mlx-Mistral-7B-Instruct--20260527-171852"

python tools/run_benchmark.py --base "local:$RUN_ID"

When it finishes, narrowhighway.com/benchmark shows the sovereign adapter alongside the Anthropic baseline.

4. Restart the engine

If /health hangs or /keep shows red, kick it. Watchdog auto-recovers after 3 missed health checks (~90 sec), but you can preempt.

# Quick path — run as Administrator
& "$env:USERPROFILE\OneDrive\Desktop\Restart_Concordance_Server.ps1"

Or from Explorer: Desktop → right-click Restart_Concordance_Server.ps1 → Run with PowerShell.

5. Watchdog & backup

Watchdog (auto-restart on hang)

# Run in its own PowerShell window — leave open
cd "$env:USERPROFILE\OneDrive\Documents\Claude\Projects\Lighthouse"
python tools\engine_watchdog.py

Polls /health every 30 sec. Three consecutive failures triggers the restart script. 5-minute cooldown to prevent restart storms.

Substrate backup (nightly tarball)

cd "$env:USERPROFILE\OneDrive\Documents\Claude\Projects\Lighthouse"
python tools\substrate_backup.py

Writes to ~/Backups/narrowhighway/. Schedule via Task Scheduler if you want it automatic.

6. Pending Mac patches

The Mac-side Claude found four small drifts when running against real-world mlx-lm 0.31+ and macOS Python 3.9.6. When the Mac sends these files via tailscale file cp, drop them into the canonical sources:

  1. tools/finetune_mlx.py — current code uses --lora-rank/--lora-alpha flags that mlx-lm 0.31+ moved to a YAML config block. The Mac's patched copy has the working form.
  2. mac/START_FINETUNE.md — every python3 should be python3.10. Mac's system Python is 3.9.6 which doesn't satisfy mlx-lm.
  3. site/mac/install.sh — same python3 → python3.10 swap; document --batch-size 2 default (28-pair corpus can't sustain batch=4 in validation).
  4. mac/push-adapter.sh — prefer brew's tailscale CLI over the GUI's bundled one (GUI sandbox blocks file cp for arbitrary paths).

Receive them with: tailscale file get data/inbox/mac — then diff and apply to the four canonical files. Push the updates to site/mac/ so the next operator running the bootstrap gets the working copies.
