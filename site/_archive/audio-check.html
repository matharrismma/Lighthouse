Audio Check

Three quick tests. If any of them gives you sound, the browser/system is fine and the problem is in the video file. If none of them give you sound, the issue is browser- or system-level — see the checklist below.

Test 1 — Generated tone (no file, no streaming)

Click the button. You should hear a 1-second 440 Hz beep.
Awaiting click…

Test 2 — Inline audio element (HTML5 audio, base64-embedded)

Click play. You should hear a short ascending chime.
Awaiting click…

Test 3 — Local pilot video (streamed via engine)

Click play, then make sure the speaker icon in the video controls is NOT muted (click it to toggle). You should hear narration.
Awaiting click…

If none of the tests played sound, check this list:

  1. Browser tab mute. Right-click the browser tab itself → "Unmute site" or "Unmute tab". Chrome/Firefox can mute individual tabs and it sticks.
  2. Volume mixer (Windows). Right-click the speaker icon in the system tray → "Open Volume Mixer". Make sure your browser (Chrome/Edge/Firefox) is not muted there.
  3. Site sound permission (Chrome). Click the padlock icon in the address bar → Site Settings → Sound → set to "Allow" (some installs default to "Block").
  4. Output device. Click the speaker icon in the system tray. Make sure the correct device is selected (speakers vs Bluetooth headphones vs HDMI). If a Bluetooth headset paired but is off, Windows still routes there silently.
  5. Headphones / speakers. Are they actually plugged in and turned on?
  6. Master volume. Is the system volume above 0?

After fixing any of the above, hit Reload (Ctrl+R) on this page and run the tests again.
