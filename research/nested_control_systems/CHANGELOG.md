All notable changes to the NHANES Framework Validation pipeline.

Format: [Semantic versioning]. Each entry includes what changed, why, and

## [v3.1-L1] — 2026-03-04

### Added (L1 proxy composite)
- **L1_proxy layer in E3:** Cellular stress proxy composite using five
  NHANES-available downstream markers: RDW (LBXRDW), GGT (LBXSGTSI),
  ALT (LBXSALT), uric acid (LBXSUA), ferritin (LBXFER). Min-obs 3 of 5.
  Cycle-wise z-scored mean, direction-aligned (higher = more stress).
- **Scope caveat documented:** L1_proxy is explicitly labeled as an indirect
  approximation. Direct L1 markers (HSP70, γ-H2AX, mitophagy flux, UPR)
  are not available in NHANES. True L1 validation path is UK Biobank Olink
  proteomics. Caveat appears in code docstring, ANALYSIS_PLAN, EXTENSIONS_PLAN,
  E3 results JSON, and README.
- **Module additions:** FERTIN and INS added to MODULE_CANDIDATES in
  nhanes_validate.py (downloader will fetch FERTIN_G/H/I/J and INS_G/H/I/J).
- **compute_l1_proxy()** method added to BiomarkerExtensions class.
  Called automatically from register_extended_indicators().

### Does not require base hypothesis re-run
- L1_proxy is an E3 extension — H1/H2/H3 thresholds and definitions unchanged.
- L1_proxy is **not** added to the upstream_manifold composite (which is
  specified in the pre-registered ANALYSIS_PLAN as L5+L3+L2).

## [v3.1] — 2026-03-04

### Changed (tougher thresholds + better discrimination + bias checks)
- **PREREG_AUC_DELTA raised to 0.015** (from 0.01; tighter falsification)
- **Clinical bar (≥0.02) now gates SUPPORTED verdict** (not just reported)
- **Brier score + partial AUC (max_fpr=0.2)** added to H1 and H3
  (AUC alone is insensitive to calibration)
- **Simulation reps raised to 2000** default (from 500; tighter estimates)
- **Dual-β simulation**: true_hierarchy (β=0.3) + true_hierarchy_weak (β=0.2)
  to map power curve as function of direct effect size
- **Collider bias check** added to E1 causal module: tests whether L5→outcome
  coefficient inflates or reverses when conditioning on L2 (a potential
  common effect / collider)
- **Competing-risks analysis** added to E2 survival: cause-specific hazards
  (CVD vs cancer vs other) to test metabolic manifold specificity
- **Time-varying mediation** added to E6: cycle-stratified ACME stability
  check, sign consistency, proportion CV, and longitudinal readiness
  framework for multi-wave cohorts
- **Factor stability cross-validation** added to E5: split-half Tucker's
  congruence, bootstrap congruence (50 reps), and layer alignment stability

### Requires re-run
- ALL hypotheses must be re-evaluated (threshold change)

## [v3.0] — 2026-03-04

### Added
- **E1 — Causal Inference:** IPW and g-computation for L5→L2/L3 causal
  effects, negative control outcomes to detect residual confounding,
  bootstrap CIs for all causal estimates
- **E2 — Incident Events:** Kaplan-Meier by manifold tertile, log-rank
  test, per-layer hazard estimation via logistic approximation
- **E3 — Biomarker Extensions:** Plugin registry for HOMA-IR, GGT,
  adipokines, vitamin D, folate, ferritin; auto-registers into existing
  or new layer composites
- **E4 — External Cohort Framework:** Abstract CohortAdapter with variable
  mapping tables for UK Biobank, All of Us; replication checklist
- **E5 — Manifold Learning:** Factor analysis with layer alignment check,
  UMAP/t-SNE embedding with silhouette scoring, shallow autoencoder with
  latent-layer correlation analysis
- **E6 — Mediation Analysis:** Bootstrap causal mediation (ACME/ADE) for
  L5→L2→outcome and L5→L3→outcome paths, sequential mediation,
  proportion mediated estimation
- **E7 — Prospective Simulations:** True hierarchy / null / reverse
  scenarios × 500 reps, power and FPR calibration, visualization
- **E8 — Subgroup Analysis:** Discordance profiles (L5-high/L2-low etc.),
  disease prevalence characterization, interaction tests for L5 predictive
  value by subgroup
- **E9 — Registered Report:** Adversarial predictions, replication
  checklist, Dockerfile for containerized reproduction

### Files
- nhanes_extensions.py (new, 1690 lines)
- EXTENSIONS_PLAN.md (new)
- Dockerfile (generated by E9)

### Requires
- v2.0 base pipeline must run first (produces nhanes_processed.csv)
- Extensions are additive — base results are preserved

## [v2.0] — 2026-03-03

### Added
- **Chunk 0 — Governance:** ANALYSIS_PLAN.md, DECISION_RULES.md, run_manifest.json
  emitted every run (script hash, config snapshot, dependency versions, timestamp)
- **Chunk 1 — Measurement truth:** Min-obs gating for layer composites,
  mean-of-z instead of sum-of-z, NHANES special code recoding (7/9/77/99 → NaN),
  per-layer missingness eligibility report
- **Chunk 2 — Mortality correctness:** H1 cohort eligibility enforcement,
  censoring distribution reporting, INCONCLUSIVE status for insufficient events
- **Chunk 3 — Survey weight sensitivity:** Combined 8-year weights (WTMEC2YR/4,
  WTSAF2YR/4), weighted AUC/logistic regression, --mode flag (prereg|sensitivity)
- **Chunk 4 — Design-consistent uncertainty:** Cluster bootstrap resampling
  PSUs within strata, bootstrap CIs for all hypothesis AUCs, pass-probability
  under resampling
- **Chunk 5 — H2 stability suite:** Bootstrap ARI for cluster stability,
  cycle-consistency check for cluster profiles, multi-seed GMM, additional
  external validator (A1c distribution)
- **Chunk 6 — H3 robustness suite:** Minimum cycle N for sign check (300),
  per-cycle delta distribution, meta-analytic summary, sensitivity to min-obs
  thresholds
- **Chunk 7 — Decision rubric:** Three-outcome verdict (SUPPORTED / FALSIFIED /
  INCONCLUSIVE), summary.json and summary.md with gate-by-gate reporting

### Changed
- Layer composites: sum → mean of observed z-scores
- Layer composites: NaN when below min-obs threshold (was: deflated score)
- SMQ020: values other than 1/2 → NaN (was: non-1 → 0, treating refused as "no")
- DPQ020/DPQ030: values 7/9 → NaN (was: included in sum)
- H1 outcome: INCONCLUSIVE when events < 50 (was: "fails")
- H3 cycle check: requires N ≥ 300 per cycle (was: N ≥ 200)
- H2: requires stability (ARI ≥ 0.40) + cycle consistency (was: clustering + chi-sq only)

### Requires
- Full re-run from scratch (data reprocessing due to composite changes)
